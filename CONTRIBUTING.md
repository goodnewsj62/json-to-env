# Contributing

Thanks for helping improve **json-to-env**. This project stays small and focused; contributions that match existing style and scope are easiest to review.

## Before you start

- **Bugs or ideas:** open an [issue](https://github.com/goodnewsj62/json-to-env/issues) with a short description, expected vs actual behavior, and a minimal example if it’s a bug.
- **Large changes:** briefly outline the approach in an issue first so we can align on direction.

## Pull requests

1. **One topic per PR** — easier to review and revert if needed.
2. **Match the codebase** — same patterns, naming, and TypeScript style as surrounding files. No drive-by refactors unrelated to your change.
3. **Tests** — add or update tests in `test/` for behavior changes or fixes. Run `npm test` before opening the PR.
4. **Build** — run `npm run typecheck` and `npm run build` when you touch source or types.

## What we look for

- Clear commit messages (what changed and why, in plain language).
- Backward compatibility unless the change is explicitly breaking and documented.

## Code of conduct

Be respectful in issues and PRs. Assume good intent and keep feedback specific and constructive.
